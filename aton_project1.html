Introduction

For my first project, I will be looking at the Human Microbiome Project (iHMP) dataset. I pulled the data from Qiita, and performed some initial processing which is beyond the scope of this project, but if you are curious you can find the basics in the "Linux Commands for Data Processing" document, which I put together for the lab I work in. The dataset contains multiple files which cover 1,569 patients in total. The data file contains metagenomic microbiome data obtained by sequencing fecal samples from the patients. The metadata file contains a few pieces of information about each patient including BMI, age, sex, diagnosis, and whether the patient was taking antibiotics at the time of sample collection.

For my project, I will attempt to predict the disease status of each patient by looking only at the microbiome data from their gut. So the data I will be using to train my models will consist of roughly 1,500 samples, which are the individual patients in the study. There are over 10,000 features in this dataset, with each feature representing an individual species which may be found in the human gut. This dimensionality presents unique challenges, as does the nature of microbiome data in general.

Microbiome data, and high-throughput sequencing (HTS) data in general, is considered compositional. This essentially means that the output from the sequencing machine can tell us the relative abundance of the species of bacteria present in each sample, but it can't actually tell us the total abundance of each species in the sample. This also means that the assumption of true independence of our features does not hold for HTS data.

Additionally, each persons microbiome is purportedly as unique as their fingerprints. For this project, this means that there may be features which are only found in a very small percentage of the samples. Of course, the usual method to handle this sort of problem would be imputation or something similar. However, I am going to try a very simple, possibly crude, but potentially very effective method of feature selection which I will investigate in this project.

My idea is to look only at those features which are present in every sample. Surprisingly, as you will see, this amounts to a very small portion of the original 10,000+ features in our data. After seeing what the performance is like when we only look at those features which are present in all samples, I will do a quick and simple look at what happens when we look at features which are in a range of percentiles of the samples.

Transformation

Here we will use counts per million (CPM). This divides the abundance of each species in a sample by the total abundance accross all species in that sample, and then multiplies it by 1 million. We can do this before doing test_train_split because the transformation is only taking place on a per sample basis. The logic here is that sample collection and sequencing runs will have different numbers of DNA sequencing reads, so we can't necessarily directly compare raw frequencies accross samples. To handle this, we divide by the sum of frequencies per sample to get the relative abundance, and then multiply by 1 million to give the numbers a reasonable scale.

Okay, but what about all the other features?

I'm curious what happens when we try to train models on different numbers of features based on the percent of samples which those features are found in. For instance, the whole notebook up till this point has been looking at only those features which are present in all samples, or 100%. What happens when we look at all the features which are in at least 60% of the samples, or 75%? Below, I wrote some code that will loop through various thresholds from 1% to 100%. These thresholds represent the percent of samples that features must be present in, in order to be kept for model training and testing. I ran it 5 times, using different random seeds for each run, in what is kind of like a weird sloppy cross-validation. The idea is that I want to get an average accross a set of models, not just look at the performance of one model with one random seed.

In[59]:
threshs = range(1, 101, 3)
acc_total = []
noise = [1, 2, 3, 4, 5]

for i in noise:
    acc = []
    for j in threshs:
        # get the feature tables
        th = np.percentile(num_samples_present, j)
        features_to_keep = num_samples_present[num_samples_present >= th].index
        data_revised = data.loc[:, features_to_keep]
    
        # perform transformation
        data_revised = data_revised.T
        data_cpm = (data_revised / data_revised.sum()) * 1000000
        data_cpm = data_cpm.T
    
        # test_train_split
        X = data_cpm
        y = meta['diagnosis_num'].values.ravel()
        X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42*i)
    
        # instantiate model
        model = RandomForestClassifier(random_state=42*i)
    
        # train model
        model.fit(X_train, y_train)
    
        # predict on test data
        preds = model.predict(X_test)
    
        # get accuracy score
        acc.append(accuracy_score(y_test, preds))
    
    acc_total.append(acc)
